What is a robots.txt file?

A robots.txt file is a text file that resides in the root directory of your website. It uses the Robots Exclusion Standard to tell web robots (most often search engine crawlers like Googlebot) which pages or files they can or cannot request from your site. This is mainly used to avoid overloading your site with requests or to keep certain files or directories out of search engine indexes.

Common Directives

  • User-agent: The crawler to which the rule applies (e.g., Googlebot). Using an asterisk * applies the rules to all crawlers.
  • Disallow: A directory or page you want to block the crawler from accessing.
  • Allow: A directory or page you want to allow the crawler to access. This is especially useful for allowing access to a specific file within a disallowed directory.
  • Sitemap: The absolute URL to your website's XML sitemap, helping search engines discover your pages more efficiently.
  • Crawl-delay: The number of seconds a crawler should wait between requests to your server. (Note: Googlebot does not support this directive, but others like Bingbot do).

Best Practices

  • Always place your robots.txt file in the top-level directory of your website (e.g., https://example.com/robots.txt).
  • Use the * User-agent to set general rules for all crawlers, and then add specific rules for individual crawlers if needed. Specific rules override general ones.
  • Never use robots.txt to hide sensitive data. If you have confidential information, use password protection or other server-level security instead.
  • Ensure you don't accidentally block CSS, JavaScript, or image files that are needed to render your page properly, as this can negatively impact your SEO.
