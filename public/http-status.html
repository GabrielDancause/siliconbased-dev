HTTP Status Code Reference

Search, examples, and troubleshooting for HTTP response status codes.

Understanding HTTP Status Codes

HTTP status codes are three-digit numbers returned by a server in response to a client's request made to the server. They provide a quick way to understand whether a specific HTTP request has been successfully completed, and if not, why.

Status Code Categories

Status codes are divided into five distinct classes (or categories). The first digit of the status code defines its class of response:

  • 1xx Informational responses (100–199): The request was received, continuing process. These are generally provisional responses.
  • 2xx Successful responses (200–299): The request was successfully received, understood, and accepted. (e.g., 200 OK).
  • 3xx Redirection messages (300–399): Further action needs to be taken in order to complete the request. Usually used for URL redirects.
  • 4xx Client error responses (400–499): The request contains bad syntax or cannot be fulfilled. The error is on the client's side (e.g., 404 Not Found).
  • 5xx Server error responses (500–599): The server failed to fulfill a valid request. The error is on the server's side (e.g., 500 Internal Server Error).

Troubleshooting Common HTTP Errors

When you encounter HTTP errors (4xx or 5xx), here are some general troubleshooting steps:

  • 400 Bad Request: Check the URL for typos. Ensure the request payload or syntax is properly formatted.
  • 401 Unauthorized / 403 Forbidden: Verify your authentication credentials. Check if you have the necessary permissions to access the resource.
  • 404 Not Found: Double-check the URL. The resource may have been moved or deleted. If you own the site, check your routing or rewrite rules.
  • 500 Internal Server Error: Look into your server's error logs. This is a generic catch-all for unexpected server conditions, often caused by misconfigured software or code bugs.
  • 502 Bad Gateway / 504 Gateway Timeout: These often indicate network issues between servers (e.g., a reverse proxy like Nginx failing to reach the backend application). Check the health of your upstream servers.
