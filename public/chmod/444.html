When to use chmod 444

Use 444 for: critical config files you don't want accidentally edited, published documents, templates that should never change, legal documents, and any file you want to protect from accidental modification.

Security note: 444 is very safe — it's read-only. The owner can still change the permissions back with chmod (since they own the file), so it's protection against accidents, not malicious changes.

Consider instead: If only the owner should read: use 400. If the owner needs to edit: use 644. If execute is needed: use 555.

Common Mistake

Thinking 444 prevents the owner from changing the file. The owner can always chmod the file back to writable since they own it. For true immutability, use chattr +i on Linux.

chmod 444 vs 644

444 vs 644
444 vs 644: 644 lets the owner write, 444 doesn't. Use 444 when you want to prevent accidental edits (even by yourself). Use 644 for normal files you actively maintain.

Frequently Asked Questions

Can the owner still delete a 444 file?
It depends on the directory permissions, not the file permissions. If the owner has write permission on the parent directory, they can delete the file even though it's 444.
How do I edit a 444 file?
Change the permissions first: chmod 644 filename, edit it, then chmod 444 filename to re-protect it. Or use sudo/root to override.
Is there a stronger protection than 444?
Yes. On Linux, chattr +i filename makes a file truly immutable — even root can't modify or delete it without first running chattr -i. This is stronger than any chmod setting.
