Understanding Font Size Units

Choosing the right CSS unit for typography is crucial for building accessible and responsive web designs. Here is a breakdown of the most common units:

Pixels (px)

Pixels are absolute units. One pixel represents one dot on a screen (historically, though modern high-DPI screens abstract this). While px is precise, it does not scale with user accessibility settings. If a user increases their browser's default font size, text set in px will not adapt.

Rem (Root Em)

Rem is a relative unit based on the root element's (<html>) font size. By default in most browsers, 1rem equals 16px. Because it respects the user's base font size settings, rem is widely considered the best unit for web accessibility. If a visually impaired user sets their base size to 24px, your 2rem heading scales gracefully to 48px.

Em

Em is also a relative unit, but it is based on the font size of its immediate parent element. This can cause compounding effects. For example, if a list is set to 1.2em, and contains a nested list also set to 1.2em, the nested text will actually be 1.44em relative to the root. Ems are best used for margins and padding that need to scale directly with the local text size, rather than for typography itself.

Points (pt)

Points are an absolute unit primarily used in print media. 1 point is exactly 1/72 of an inch. While you can use pt in CSS, it is generally recommended only for print stylesheets (e.g., @media print), as screen rendering of physical units can be unpredictable across different devices.

Viewport Width (vw)

Viewport Width is a relative unit representing 1% of the browser's total width. 10vw means 10% of the screen width. While great for massive hero typography that needs to fluidly scale down on mobile, it can cause text to become unreadably small on narrow screens or excessively large on ultrawide monitors. It is often combined with clamp() in modern CSS to provide safe minimum and maximum boundaries.
