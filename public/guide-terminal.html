Terminal Commands Cheat Sheet

Essential terminal and bash commands for macOS and Linux.

Understanding the Command Line

The command line interface (CLI) is a powerful tool that allows you to interact directly with your computer's operating system using text commands. While graphical user interfaces (GUIs) are user-friendly, the terminal offers unparalleled speed, automation capabilities, and access to deep system functions.

Whether you're navigating file systems, managing server processes, or configuring network settings, mastering these essential terminal commands is a critical step for developers, system administrators, and power users.

macOS vs. Linux Commands

Both macOS and most Linux distributions share a common ancestry rooted in Unix. Because of this, the vast majority of terminal commands are identical across both platforms. The default shell on modern macOS is `zsh` (Z shell), while many Linux distributions default to `bash` (Bourne Again SHell). Both shells support the standard commands listed in this cheat sheet.

  • Package Managers: Linux users typically use apt (Debian/Ubuntu) or yum/dnf (RedHat), while macOS users rely on third-party managers like Homebrew (brew).
  • Clipboard Access: macOS has built-in pbcopy and pbpaste commands for clipboard integration. On Linux, you would typically install and use xclip or xsel.
  • Networking Tools: Some legacy networking tools like ifconfig are still standard on macOS but have been deprecated in favor of the ip command suite on modern Linux systems.

Tips for Terminal Efficiency

  • Use the Tab Key: Pressing Tab will auto-complete file names, directory paths, and even commands, saving you time and preventing typos.
  • Command History: Use the Up and Down arrow keys to cycle through your previous commands. You can also use Ctrl + R to reverse search through your command history.
  • Stopping Processes: If a command is running and you need to stop it immediately, press Ctrl + C to send an interrupt signal.
  • Chaining Commands: You can run multiple commands sequentially by separating them with && (e.g., mkdir build && cd build). The second command will only run if the first one succeeds.
