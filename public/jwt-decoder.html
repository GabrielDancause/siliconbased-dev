Decoding JSON Web Tokens (JWT)

JSON Web Tokens are an open, industry standard RFC 7519 method for representing claims securely between two parties. They are widely used for authentication and authorization in modern web applications. Because JWTs are compact and URL-safe, they can be easily transmitted via HTTP headers, query parameters, or inside cookies.

Common JWT Security Pitfalls

  • Assuming JWTs are encrypted: By default, JWTs are only encoded using base64url. Anyone who intercepts the token can decode and read the payload using a jwt decoder. Never put sensitive data (like passwords) in a JWT payload.
  • None Algorithm Vulnerability: Some poorly implemented JWT libraries accept tokens where the alg header is set to none. Attackers can create tokens with elevated privileges, set alg: none, and bypass signature verification completely.
  • Not verifying the signature: A token is only trustworthy if its signature is validated against your server's secret or public key.
  • Ignoring expiration (exp): Tokens should be short-lived. Always check the exp claim to ensure the token hasn't expired. This tool automatically flags expired tokens for you.

Standard Claims Explained

The JWT specification defines a set of registered claims. While they are not mandatory, they provide a standard way to convey specific information:

  • iss (Issuer): Identifies the principal that issued the JWT.
  • sub (Subject): Identifies the subject of the JWT (often the user ID).
  • aud (Audience): Identifies the intended recipients of the JWT.
  • exp (Expiration Time): The exact timestamp when the token expires and MUST NOT be accepted.
  • nbf (Not Before): The time before which the token MUST NOT be accepted.
  • iat (Issued At): The time the token was created.
  • jti (JWT ID): A unique identifier for the token, useful for preventing replay attacks.

Frequently Asked Questions

What is a JWT decoder and how does it work?
A JWT decoder is a tool that takes a base64url-encoded JSON Web Token and breaks it apart into its three components: header, payload, and signature. It decodes the base64url strings into readable JSON so you can inspect the data inside the token, such as user claims and expiration dates.
How to decode a JWT token?
To decode a JWT token, you simply paste the token string (which looks like xxxxx.yyyyy.zzzzz) into a JWT debugger. The tool splits the token by the dots (.) and decodes the first two parts (header and payload) from base64url format into plain JSON.
Is this a secure jwt.io alternative?
Yes, this tool is a secure jwt.io alternative. It runs entirely within your browser using client-side JavaScript. Your tokens are never transmitted to any server. You can even load the page, disconnect from the internet, and continue to use the tool completely offline.
Can I decode a JWT without a secret?
Yes, you can decode the header and payload of a JWT without knowing the secret key. The data in a JWT is only encoded (using base64url), not encrypted. However, you cannot verify the signature or ensure the token hasn't been tampered with unless you have the correct secret or public key.
Why is my JWT signature invalid in the jwt debugger?
An invalid signature usually means one of three things: the token data (header or payload) was modified after it was signed, you are using the wrong secret/public key to verify it, or you selected the wrong algorithm. Ensure your secret matches exactly what the server uses.
