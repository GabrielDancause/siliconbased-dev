cURL Commands Guide

Reference, examples, and testing patterns for API requests.

What is cURL?

cURL (Client URL) is a versatile command-line tool and library for transferring data with URLs. It supports a vast array of protocols, including HTTP, HTTPS, FTP, SFTP, and many more. It is an industry standard for developers to interact with APIs, test endpoints, download files, and automate network requests directly from the terminal.

Why Use cURL for API Testing?

While GUI tools like Postman or Insomnia are popular, cURL remains highly relevant for several reasons:

  • Ubiquity: cURL comes pre-installed on most macOS, Linux, and Windows 10/11 environments.
  • Speed: Executing a command from the terminal is incredibly fast for simple checks.
  • Automation: Easily scriptable in bash scripts and CI/CD pipelines.
  • Sharability: A single string of text can perfectly encapsulate a complex HTTP request, making it easy to share via chat or documentation. Most GUI API clients offer an "Import from cURL" feature.

Common Command Line Flags

Understanding these essential flags will cover the vast majority of use cases:

  • -X, --request: Specifies the HTTP method to use (GET, POST, PUT, DELETE, etc.). Note: cURL defaults to GET, and automatically switches to POST if you use -d.
  • -H, --header: Adds a custom header to the request (e.g., -H "Authorization: Bearer token" or -H "Content-Type: application/json"). You can use multiple -H flags.
  • -d, --data: Sends data in the request body. Automatically sets the method to POST and the Content-Type header to application/x-www-form-urlencoded unless overridden.
  • -i, --include: Includes HTTP response headers in the output. Useful for checking status codes and cookies.
  • -v, --verbose: Makes the output verbose. Shows the full request and response, including headers and the handshake. Invaluable for debugging.
  • -L, --location: If the server responds with a 3xx redirect status code, this tells cURL to redo the request to the new location.
  • -u, --user: Specifies the user name and password to use for server authentication (e.g., user:password).
  • -o, --output: Writes output to a file instead of stdout.
  • -s, --silent: Mutes cURL's progress meter and error messages. Useful for scripting.
