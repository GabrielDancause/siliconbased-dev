What is CORS?

Cross-Origin Resource Sharing (CORS) is an HTTP-header based mechanism that allows a server to indicate any origins (domain, scheme, or port) other than its own from which a browser should permit loading resources. CORS also relies on a mechanism by which browsers make a "preflight" request to the server hosting the cross-origin resource, in order to check that the server will permit the actual request.

Common CORS Headers

  • Access-Control-Allow-Origin: Indicates whether the response can be shared. Can be a specific domain or * (all domains).
  • Access-Control-Allow-Methods: Specifies the methods allowed when accessing the resource (e.g., GET, POST, OPTIONS).
  • Access-Control-Allow-Headers: Indicates which HTTP headers can be used during the actual request.
  • Access-Control-Allow-Credentials: Indicates whether the response to the request can be exposed when the credentials flag is true (e.g., cookies, authorization headers).

CORS Security Risks

Misconfiguring CORS can lead to security vulnerabilities. For example, using the wildcard * for the origin is often used for public APIs, but it should be avoided if the endpoint handles sensitive user data or requires authentication. Furthermore, the specification strictly forbids setting Access-Control-Allow-Credentials: true when the origin is a wildcard *.
