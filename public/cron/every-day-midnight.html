When to Use This

Daily backups, log archival, report generation, database maintenance, certificate renewal checks.

Crontab Entry

0 0 * * * 0 0 * * * /opt/backup/daily-backup.sh >> /var/log/backup.log 2>&1
0 0 * * * /path/to/your/script.sh

Things to Know

⚠️ Watch out

Midnight in which timezone? Cron uses the system timezone. If your server is UTC but your users are in EST, 'midnight' is 7pm their time. Always check with 'date' or 'timedatectl'.

💡 Pro tip

Avoid minute 0, hour 0 — it's the most congested cron time. Use '15 0 * * *' or '0 1 * * *' to avoid the midnight stampede. Your database backup doesn't need to start at exactly 00:00.
