HTTP Status Codes Reference

Every Code Explained with Examples

100 Continue

What it means

The server has received the request headers and the client should proceed to send the request body.

When you see it

Seen when sending large payloads where the client asks the server if it's willing to accept the request before sending it.

Example

curl -v -X POST -H 'Expect: 100-continue' https://example.com
101 Switching Protocols

What it means

The requester has asked the server to switch protocols and the server has agreed to do so.

When you see it

Commonly seen when upgrading an HTTP connection to a WebSocket connection.

Example

curl -v -i -N -H 'Connection: Upgrade' -H 'Upgrade: websocket' -H 'Sec-WebSocket-Key: SGVsbG8sIHdvcmxkIQ==' -H 'Sec-WebSocket-Version: 13' https://example.com
102 Processing

What it means

The server has received and is processing the request, but no response is available yet.

When you see it

Used in WebDAV to prevent the client from timing out when a request takes a long time to process.

Example

curl -v -X PROPFIND https://example.com
103 Early Hints

What it means

Used to return some response headers before final HTTP message.

When you see it

Useful to allow the client to start preloading resources while the server prepares the main response.

Example

curl -v https://example.com
200 OK

What it means

The request succeeded. The result meaning depends on the HTTP method.

When you see it

The standard response for successful HTTP requests (GET, POST, PUT, etc).

Example

curl -I https://example.com
201 Created

What it means

The request succeeded, and a new resource was created as a result.

When you see it

Typically the response sent after POST requests, or some PUT requests.

Example

curl -v -X POST -d 'name=test' https://example.com/api/resource
202 Accepted

What it means

The request has been received but not yet acted upon. It is noncommittal.

When you see it

Used for asynchronous processing where the server queues the request to be processed later.

Example

curl -v -X POST https://example.com/api/async-task
203 Non-Authoritative Information

What it means

The returned metadata is not exactly the same as is available from the origin server, but is collected from a local or a third-party copy.

When you see it

Used mostly by proxies or caching servers.

Example

curl -I https://example.com
204 No Content

What it means

There is no content to send for this request, but the headers may be useful.

When you see it

Often returned by DELETE requests or PUT requests that successfully update a resource without needing to return the entity.

Example

curl -v -X DELETE https://example.com/api/resource/1
205 Reset Content

What it means

Tells the user agent to reset the document which sent this request.

When you see it

Useful for clearing a form after a successful submission.

Example

curl -v -X POST https://example.com/api/form
206 Partial Content

What it means

This response code is used when the Range header is sent from the client to request only part of a resource.

When you see it

Used for resumeable downloads or streaming media.

Example

curl -v -H 'Range: bytes=0-1023' https://example.com/video.mp4
300 Multiple Choices

What it means

The request has more than one possible response. The user agent or user should choose one of them.

When you see it

Rarely used, but could be for presenting multiple video formats or language options.

Example

curl -I https://example.com
301 Moved Permanently

What it means

The URL of the requested resource has been changed permanently. The new URL is given in the response.

When you see it

Used when migrating to a new domain or restructuring a website's URLs.

How to fix it

Update client links or bookmarks to use the new URL provided in the Location header.

Example

curl -I https://example.com/old-page
302 Found

What it means

This response code means that the URI of requested resource has been changed temporarily.

When you see it

Used for temporary redirects, such as during site maintenance or A/B testing.

How to fix it

No action needed for temporary redirects, follow the Location header.

Example

curl -I https://example.com/temp-page
303 See Other

What it means

The server sent this response to direct the client to get the requested resource at another URI with a GET request.

When you see it

Often used after a successful POST request to redirect the user to a success page (Post/Redirect/Get pattern).

Example

curl -v -X POST -d 'data=1' https://example.com/form
304 Not Modified

What it means

This is used for caching purposes. It tells the client that the response has not been modified, so the client can continue to use the same cached version of the response.

When you see it

When a client sends a conditional GET request (e.g., using If-Modified-Since).

Example

curl -I -H 'If-None-Match: "etag-value"' https://example.com/resource
307 Temporary Redirect

What it means

The server sends this response to direct the client to get the requested resource at another URI with the same method that was used in the prior request.

When you see it

Similar to 302, but guarantees the HTTP method will not be changed.

Example

curl -v -X POST https://example.com/redirect
308 Permanent Redirect

What it means

The request and all future requests should be repeated using another URI. The HTTP method is not allowed to change.

When you see it

Similar to 301, but guarantees the HTTP method will not be changed.

How to fix it

Update clients to use the new URI.

Example

curl -I https://example.com/old-api
400 Bad Request

What it means

The server could not understand the request due to invalid syntax.

When you see it

Sent when the client's request is malformed, missing required parameters, or has invalid data.

How to fix it

Check the request syntax, headers, and body parameters. Ensure they match the API specification.

Example

curl -v -X POST -d 'invalid json{' https://example.com/api
401 Unauthorized

What it means

Although the HTTP standard specifies 'unauthorized', semantically this response means 'unauthenticated'. That is, the client must authenticate itself to get the requested response.

When you see it

When trying to access a protected resource without valid authentication credentials.

How to fix it

Provide valid authentication credentials (e.g., a Bearer token or API key) in the request headers.

Example

curl -v -H 'Authorization: Bearer invalid_token' https://example.com/api/protected
402 Payment Required

What it means

This response code is reserved for future use. The initial aim for creating this code was using it for digital payment systems.

When you see it

Rarely used, but sometimes implemented by APIs to indicate a quota limit or required subscription.

How to fix it

Check your billing status or API usage quotas.

Example

curl -I https://api.example.com/premium-data
403 Forbidden

What it means

The client does not have access rights to the content; that is, it is unauthorized, so the server is refusing to give the requested resource.

When you see it

When authenticated but lacking permissions to perform an action or access a specific resource.

How to fix it

Ensure the authenticated user has the necessary roles or permissions to access the resource. Check server configuration (e.g., directory indexing disabled).

Example

curl -I https://example.com/admin
404 Not Found

What it means

The server can not find the requested resource. In the browser, this means the URL is not recognized.

When you see it

When requesting a page or API endpoint that does not exist.

How to fix it

Verify the URL path is correct. If it's an API, check if the resource ID exists.

Example

curl -I https://example.com/does-not-exist
405 Method Not Allowed

What it means

The request method is known by the server but is not supported by the target resource.

When you see it

When making a POST request to an endpoint that only accepts GET requests, for example.

How to fix it

Change the HTTP method to one supported by the endpoint (e.g., from POST to GET). Check the 'Allow' header in the response.

Example

curl -v -X PUT https://example.com/api/read-only
406 Not Acceptable

What it means

This response is sent when the web server, after performing server-driven content negotiation, doesn't find any content that conforms to the criteria given by the user agent.

When you see it

When the Accept header asks for a format the server cannot provide (e.g., asking for application/xml when only application/json is available).

How to fix it

Modify the 'Accept' header to a format the server supports.

Example

curl -v -H 'Accept: application/pdf' https://example.com/api/data
407 Proxy Authentication Required

What it means

This is similar to 401 Unauthorized but authentication is needed to be done by a proxy.

When you see it

When a client must first authenticate itself with the proxy.

How to fix it

Provide proxy credentials via the Proxy-Authenticate header.

Example

curl -v -x http://proxy.example.com:8080 https://example.com
408 Request Timeout

What it means

This response is sent on an idle connection by some servers, even without any previous request by the client.

When you see it

When the server timed out waiting for the request.

How to fix it

Try sending the request again or check your network connection for slowness.

Example

curl -v https://example.com
409 Conflict

What it means

This response is sent when a request conflicts with the current state of the server.

When you see it

Often used when trying to create a resource that already exists (e.g., duplicate email address).

How to fix it

Resolve the conflict, such as choosing a different username or updating the existing resource.

Example

curl -v -X POST -d 'username=existing' https://example.com/api/users
410 Gone

What it means

This response is sent when the requested content has been permanently deleted from server, with no forwarding address.

When you see it

Used for resources that were intentionally removed and will not return.

How to fix it

Remove references to the resource. It is intentionally unavailable.

Example

curl -I https://example.com/deleted-item
411 Length Required

What it means

Server rejected the request because the Content-Length header field is not defined and the server requires it.

When you see it

When uploading data without specifying its length.

How to fix it

Include a valid 'Content-Length' header in your request.

Example

curl -v -X POST -H 'Transfer-Encoding: chunked' https://example.com/upload
412 Precondition Failed

What it means

The client has indicated preconditions in its headers which the server does not meet.

When you see it

When conditionally updating a resource (e.g., with If-Match) and the condition fails.

How to fix it

Check your precondition headers (If-Match, If-Unmodified-Since) and ensure you have the latest state.

Example

curl -v -X PUT -H 'If-Match: "old-etag"' https://example.com/resource
413 Payload Too Large

What it means

Request entity is larger than limits defined by server.

When you see it

When uploading a file that exceeds the maximum allowed size.

How to fix it

Reduce the size of the request payload or configure the server to accept larger limits.

Example

curl -v -X POST -F 'file=@hugefile.zip' https://example.com/upload
414 URI Too Long

What it means

The URI requested by the client is longer than the server is willing to interpret.

When you see it

When sending too much data in a GET request query string.

How to fix it

Convert the GET request to a POST request and put the data in the body.

Example

curl -I https://example.com/?very_long_query_string...
415 Unsupported Media Type

What it means

The media format of the requested data is not supported by the server.

When you see it

When sending XML to an endpoint that only accepts JSON.

How to fix it

Change the Content-Type header and the format of your payload to one supported by the server.

Example

curl -v -X POST -H 'Content-Type: text/plain' -d 'data' https://example.com/api
416 Range Not Satisfiable

What it means

The range specified by the Range header field in the request cannot be fulfilled.

When you see it

When requesting a part of a file that is outside its bounds.

How to fix it

Ensure the requested byte range is within the actual size of the resource.

Example

curl -v -H 'Range: bytes=100000-200000' https://example.com/smallfile.txt
417 Expectation Failed

What it means

This response code means the expectation indicated by the Expect request header field cannot be met by the server.

When you see it

When using the Expect: 100-continue header and the server refuses.

How to fix it

Remove the Expect header or adjust the request so the server accepts it.

Example

curl -v -H 'Expect: 100-continue' https://example.com
418 I'm a teapot

What it means

The server refuses the attempt to brew coffee with a teapot.

When you see it

An Easter egg from the April Fools' Hyper Text Coffee Pot Control Protocol (HTCPCP).

How to fix it

Brew coffee with a coffee maker instead.

Example

curl -I https://example.com/brew
422 Unprocessable Entity

What it means

The request was well-formed but was unable to be followed due to semantic errors.

When you see it

Common in APIs when validation fails (e.g., providing a string for an integer field).

How to fix it

Fix the validation errors in the request body as indicated by the server.

Example

curl -v -X POST -d '{"age": "twenty"}' https://example.com/api/users
425 Too Early

What it means

Indicates that the server is unwilling to risk processing a request that might be replayed.

When you see it

When using TLS Early Data (0-RTT).

How to fix it

Retry the request on a fully established connection.

Example

curl -v https://example.com
426 Upgrade Required

What it means

The server refuses to perform the request using the current protocol but might be willing to do so after the client upgrades to a different protocol.

When you see it

When the server requires TLS/SSL or a newer HTTP version.

How to fix it

Upgrade your protocol, such as switching from HTTP/1.1 to HTTP/2.

Example

curl -I https://example.com
428 Precondition Required

What it means

The origin server requires the request to be conditional.

When you see it

Intended to prevent 'lost update' problems.

How to fix it

Include conditional headers like If-Match with your request.

Example

curl -v -X PUT https://example.com/resource
429 Too Many Requests

What it means

The user has sent too many requests in a given amount of time (rate limiting).

When you see it

When you exceed the API rate limit quota.

How to fix it

Slow down your requests. Check the Retry-After header for how long to wait before retrying.

Example

curl -I https://api.example.com/data
431 Request Header Fields Too Large

What it means

The server is unwilling to process the request because its header fields are too large.

When you see it

When sending too many cookies or excessively large custom headers.

How to fix it

Reduce the size of your request headers or clear unnecessary cookies.

Example

curl -v -H 'Large-Header: ...' https://example.com
451 Unavailable For Legal Reasons

What it means

The user agent requested a resource that cannot legally be provided, such as a web page censored by a government.

When you see it

When access is blocked due to legal demands.

How to fix it

The resource is legally blocked. Access may require geographic circumvention, which may be illegal.

Example

curl -I https://example.com/blocked
500 Internal Server Error

What it means

The server has encountered a situation it does not know how to handle.

When you see it

A generic catch-all for server-side errors, like unhandled exceptions in code.

How to fix it

Check server logs to identify the root cause of the crash or exception.

Example

curl -I https://example.com
501 Not Implemented

What it means

The request method is not supported by the server and cannot be handled.

When you see it

When requesting a method like PATCH on a server that doesn't support it.

How to fix it

Ensure you are using standard HTTP methods. If you manage the server, implement the required method.

Example

curl -X UNKNOWN_METHOD https://example.com
502 Bad Gateway

What it means

This error response means that the server, while working as a gateway to get a response needed to handle the request, got an invalid response.

When you see it

Often seen when a reverse proxy (like Nginx) cannot communicate with the upstream backend application (like Node.js or Python).

How to fix it

Check if the upstream application server is running and accessible by the proxy.

Example

curl -I https://example.com
503 Service Unavailable

What it means

The server is not ready to handle the request. Common causes are a server that is down for maintenance or that is overloaded.

When you see it

During planned maintenance windows or severe traffic spikes.

How to fix it

Wait and try again later. Check the Retry-After header. If you are the admin, scale your resources.

Example

curl -I https://example.com
504 Gateway Timeout

What it means

This error response is given when the server is acting as a gateway and cannot get a response in time.

When you see it

When the backend application takes too long to process a request and the proxy times out.

How to fix it

Optimize backend queries or increase the timeout limits on your proxy/load balancer.

Example

curl -I https://example.com
505 HTTP Version Not Supported

What it means

The HTTP version used in the request is not supported by the server.

When you see it

Rarely seen, unless using a very old or unsupported HTTP protocol version.

How to fix it

Upgrade your HTTP client.

Example

curl --http1.0 https://example.com
506 Variant Also Negotiates

What it means

The server has an internal configuration error: the chosen variant resource is configured to engage in transparent content negotiation itself, and is therefore not a proper end point in the negotiation process.

When you see it

Server misconfiguration.

How to fix it

Fix the server's content negotiation configuration.

Example

curl -I https://example.com
507 Insufficient Storage

What it means

The method could not be performed on the resource because the server is unable to store the representation needed to successfully complete the request.

When you see it

When the server runs out of disk space or database quota.

How to fix it

Free up disk space or increase the storage allocation on the server.

Example

curl -v -X POST -d 'data' https://example.com
508 Loop Detected

What it means

The server detected an infinite loop while processing the request.

When you see it

Typically associated with WebDAV.

How to fix it

Fix the loop in the directory structure or redirect rules.

Example

curl -I https://example.com
511 Network Authentication Required

What it means

Indicates that the client needs to authenticate to gain network access.

When you see it

When trying to access the internet through a captive portal (like a public Wi-Fi login screen).

How to fix it

Complete the captive portal login process.

Example

curl -I http://example.com

Frequently Asked Questions

What are HTTP status codes?

HTTP status codes are standard responses given by web servers on the internet. They indicate whether a specific HTTP request has been successfully completed, providing crucial context for troubleshooting. They are divided into five categories: informational (1xx), successful (2xx), redirection (3xx), client error (4xx), and server error (5xx).

Where can I find a complete HTTP status code list?

You can find a complete, searchable list of all standard HTTP status codes right here on this page. We've compiled every official status code from 100 to 511, including explanations, common causes, fixes for error codes, and curl examples to test them.

What does a 403 Forbidden status code mean?

A 403 Forbidden error means that the server understands your request, but refuses to authorize it. Unlike a 401 Unauthorized code (which means you need to log in), a 403 means you are authenticated, but you do not have the required permissions to access that specific page or resource.

How do I fix an HTTP 502 Bad Gateway error?

An HTTP 502 Bad Gateway error occurs when a server acting as a proxy or gateway receives an invalid response from an upstream server. To fix it, you should verify that the backend application (like Node.js, Python, or PHP) is running correctly and that the proxy server (like Nginx or Apache) is properly configured to communicate with it.

What is the difference between a 301 and 302 redirect?

A 301 Moved Permanently indicates that a resource has been permanently moved to a new URL, and search engines should update their links. A 302 Found is a temporary redirect, signaling that the resource is temporarily located elsewhere, but future requests should still use the original URL.
